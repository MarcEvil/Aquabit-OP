# 💧 AquaBit OP - Gestión Técnica de Terreno

Sistema especializado para el registro y documentación de actividades técnicas en terreno. Esta aplicación permite la captura de evidencia fotográfica y la gestión de estados de servicio de forma eficiente y centralizada.

## 🚀 Funcionalidades Principales

* **Registro Multimodal:** Documentación de procesos de Corte y Reposición con diferenciación visual por pestañas.
* **Captura de Evidencia:** Integración directa con la cámara del dispositivo para múltiples capturas simultáneas.
* **Interfaz de Usuario (UI):** Diseño neumórfico optimizado para alta visibilidad y ergonomía en dispositivos móviles.
* **Gestión de Datos:** Sincronización automática de registros con almacenamiento en la nube y base de datos relacional.
* **Auditoría Técnica:** Galería integrada para la consulta de registros históricos y verificación de servicios realizados.
* **Optimización de Recursos:** Sistema de limpieza automática de archivos para mantener la eficiencia del almacenamiento.

## 🛠️ Stack Tecnológico

* **Frontend:** HTML5, CSS3 con variables dinámicas, Tailwind CSS.
* **Backend:** Entorno de ejecución Node.js (API REST).
* **Almacenamiento:** Cloud Storage para archivos binarios y SQL para metadatos.
* **Iconografía:** Set de iconos vectoriales para interfaces técnicas.

## 📁 Estructura del Software

* `index.html`: Módulo principal de operación técnica.
* `galeria.html`: Módulo de consulta de base de datos total.
* `style.css`: Definición de estilos, temas dinámicos y efectos visuales.
* `script.js`: Lógica de negocio, controladores de eventos y comunicación con servicios externos.

## ⚙️ Notas de Seguridad y Configuración

> [!IMPORTANT]
> Por razones de seguridad, las credenciales de acceso a los servicios de nube, las API Keys y las URLs de los servidores de producción **no deben incluirse** en este documento. 
> 
> Para la configuración en entornos locales o de despliegue, asegúrese de utilizar variables de entorno o un archivo de configuración protegido para las constantes de conexión en `script.js`.

## 📱 Flujo de Trabajo

1.  **Inicio de Sesión/Operación:** Selección de la categoría de servicio mediante los botones de acción rápida.
2.  **Documentación:** Captura de fotografías y redacción de observaciones técnicas.
3.  **Sincronización:** Envío de datos al repositorio central mediante el botón de guardado.
4.  **Verificación:** Consulta de estados en el historial inferior o en la galería de base de datos.
